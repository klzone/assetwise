# Step6 - UI CSS 缺陷清单与修复建议

目标
- 统一全局样式，建立可回归的视觉基线，降低后续回归成本。

缺陷清单（初步）
- 全局变量与主题 token 不一致
  - 变量覆盖在部分组件未正确应用，暗黑模式对比度在某些区域不足。
- 间距与字体不一致
  - 列表、表单字段、按钮等交互区域的间距和字体大小在不同页面存在偏差。
- 组件变体未统一
  - 多处页面使用局部样式，未统一成统一的 Tailwind 变体或组件类。
- 颜色对比与图标色
  - 某些按钮/标签在暗黑模式下颜色对比偏低，需调整颜色变量。
- 覆盖冲突
  - 部分页面样式被局部样式覆盖，导致样式优先级难以维护。

优先修复建议
- 以 globals.css 和 tailwind.config.js 为入口，确保颜色、间距、圆角、字体的设计 Token 完整且一致地应用到所有核心组件。
- 将资产列表、侧边栏、登录页、仪表盘等高频页面落地为统一的组件变体，避免再次出现局部样式。
- 启用简单的视觉回归基线：对 3-5 个核心页面添加 baseline 截图。

产出物
- ui-css-defects.md（当前缺陷清单与复现步骤）
- baseline_screenshots/ 相关基线截图集合（初步版本）
- 扩展的 UI 测试用例草案（覆盖颜色对比、聚焦可见性、暗黑模式切换等）

后续计划
- 基于缺陷清单提交修复摘要，更新 Step7 的执行计划并推进视觉回归的落地。
