# AssetWise 数据同步功能测试指南

## 概述

AssetWise 数据同步功能提供了完整的云端与本地数据同步解决方案，包括离线支持、冲突解决、实时同步等核心功能。本指南将帮助您了解如何测试和验证这些功能。

## 功能特性

### 🔄 核心同步功能
- **双向数据同步**: 支持云端与本地数据的双向同步
- **离线队列管理**: 离线时自动缓存操作，联网后自动同步
- **实时数据更新**: 基于Supabase实时订阅的即时数据同步
- **冲突检测与解决**: 智能检测数据冲突并提供解决方案
- **数据一致性检查**: 定期验证云端与本地数据的一致性

### 📊 监控与诊断
- **同步状态监控**: 实时显示同步状态和进度
- **性能指标追踪**: 监控同步性能、内存使用、网络延迟等
- **错误日志记录**: 详细记录同步过程中的错误和警告
- **数据统计分析**: 提供同步成功率、吞吐量等统计信息

## 测试环境设置

### 1. 环境要求
- Node.js >= 18.0.0
- pnpm >= 8.0.0
- Supabase 项目配置
- 网络连接（用于测试在线/离线切换）

### 2. 安装依赖
```bash
cd assetwise-app
pnpm install
```

### 3. 环境配置
确保 `.env.local` 文件包含正确的 Supabase 配置：
```env
NEXT_PUBLIC_SUPABASE_URL=your_supabase_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key
```

## 测试用例说明

### 1. 数据库连接测试
- **目的**: 验证与Supabase数据库的连接是否正常
- **测试内容**: 执行简单的数据库查询操作
- **预期结果**: 成功连接并返回数据

### 2. 基本数据同步测试
- **目的**: 验证基本的数据同步功能
- **测试内容**: 创建、更新、删除资产数据并验证同步
- **预期结果**: 本地操作能正确同步到云端

### 3. 离线数据队列测试
- **目的**: 验证离线时的数据缓存和队列管理
- **测试内容**: 模拟离线状态下的数据操作
- **预期结果**: 离线操作被正确缓存，联网后自动同步

### 4. 数据冲突解决测试
- **目的**: 验证数据冲突的检测和解决机制
- **测试内容**: 创建数据冲突场景并测试解决方案
- **预期结果**: 冲突被正确检测并提供解决选项

### 5. 实时同步测试
- **目的**: 验证实时数据同步功能
- **测试内容**: 测试Supabase实时订阅功能
- **预期结果**: 云端数据变化能实时反映到本地

### 6. 数据一致性检查测试
- **目的**: 验证数据一致性检查功能
- **测试内容**: 比较云端与本地数据的一致性
- **预期结果**: 能正确识别数据不一致的情况

### 7. 错误处理测试
- **目的**: 验证各种错误情况的处理
- **测试内容**: 模拟网络错误、数据验证错误等
- **预期结果**: 错误被正确捕获和处理

### 8. 性能压力测试
- **目的**: 验证系统在高负载下的性能表现
- **测试内容**: 批量数据操作和同步
- **预期结果**: 满足性能要求（>10条/秒）

### 9. 网络中断恢复测试
- **目的**: 验证网络中断后的恢复机制
- **测试内容**: 模拟网络中断和恢复
- **预期结果**: 网络恢复后能自动重新同步

### 10. 大数据量同步测试
- **目的**: 验证大量数据的同步能力
- **测试内容**: 同步1000+条记录
- **预期结果**: 能稳定处理大量数据同步

## 性能基准

### 同步性能指标
- **平均同步时间**: < 5秒
- **数据吞吐量**: > 10条记录/秒
- **成功率**: > 95%
- **内存使用**: < 200MB
- **CPU使用**: < 50%

### 网络性能指标
- **网络延迟**: < 500ms
- **错误率**: < 5%
- **重试成功率**: > 90%

## 故障排除

### 常见问题

1. **数据库连接失败**
   - 检查Supabase配置是否正确
   - 验证网络连接
   - 确认API密钥有效性

2. **同步速度慢**
   - 检查网络连接质量
   - 查看性能监控指标
   - 考虑批量操作优化

3. **数据冲突频繁**
   - 检查时间戳同步
   - 优化冲突解决策略
   - 考虑增加版本控制

4. **离线同步失败**
   - 检查本地存储空间
   - 验证队列管理逻辑
   - 查看错误日志

### 调试技巧

1. **启用详细日志**
```javascript
// 在浏览器控制台中启用调试模式
localStorage.setItem('debug', 'sync:*');
```

2. **查看网络请求**
- 使用浏览器开发者工具的Network标签
- 监控Supabase API调用
- 检查WebSocket连接状态

3. **检查本地存储**
```javascript
// 查看本地存储的同步队列
console.log(localStorage.getItem('sync_queue'));
```

## 技术架构

### 同步服务架构
```
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│   前端应用      │    │   同步服务      │    │   Supabase     │
│                 │    │                 │    │                 │
│ - React组件     │◄──►│ - SyncService   │◄──►│ - PostgreSQL   │
│ - 状态管理      │    │ - 队列管理      │    │ - 实时订阅     │
│ - 本地存储      │    │ - 冲突解决      │    │ - API接口      │
└─────────────────┘    └─────────────────┘    └─────────────────┘
```

### 数据流向
```
本地操作 → 验证 → 队列 → 同步服务 → Supabase → 实时通知 → 本地更新
```
