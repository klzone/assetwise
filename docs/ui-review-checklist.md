# UI 评审清单（AssetWise）

适用范围：全站 UI 重构提交（页面与组件）。目标：统一视觉与交互，确保明/暗主题一致性与三端适配质量。

## 1. 视觉与品牌基调
- [ ] 基调“极简 + 轻新拟态”，视觉克制、信息对比明确
- [ ] 使用柔和中性底色（明：暖灰；暗：冷灰蓝）
- [ ] 点缀色为亮色（电光青/酸柠绿/品红），使用频率适度
- [ ] 图片/插画/空态风格一致（等线几何、柔性渐变）

## 2. 主题与语义色
- [ ] 明/暗双主题变量齐全（背景、前景、边框、卡片、弹层）
- [ ] 语义色覆盖：primary/secondary/accent/muted/info/success/warning/destructive
- [ ] 对比度符合 AA（文本与交互元素至少 4.5:1）
- [ ] Focus 可见，状态过渡细腻（150–240ms）

## 3. 布局与栅格
- [ ] 栅格：桌面 12 列（32px 槽），平板 8 列（24px），移动 4 列（16px）
- [ ] 基线：8pt 间距体系（4/8/12/16/24/32/48）
- [ ] 顶部工具栏 + 左侧主导航（桌面）/底部标签栏（移动）
- [ ] 红线检测：元素对齐、留白均衡、信息层级清晰

## 4. 组件与交互
- [ ] 按钮/输入/选择/开关/对话框/抽屉/Toast 等风格统一
- [ ] 圆角层级：12/16/20；新拟态弱浮层与轻内阴影在卡片/控件适度使用
- [ ] 悬浮/按下/禁用/加载各状态样式齐备
- [ ] 表格：密度可切、行悬浮显控件、批量操作明显

## 5. 表单与校验
- [ ] Label/帮助文本/错误提示可读且一致
- [ ] 焦点环覆盖全部可交互控件，键盘可达
- [ ] 错误状态具备语义色与图标指示

## 6. 导航与信息架构
- [ ] 顶栏含搜索/主题切换/全局操作，位置一致
- [ ] 侧边/底部导航条在不同端适配良好
- [ ] 面包屑与分组标题指引路径清晰

## 7. 图表与可视化
- [ ] 色板统一（主跳色 + 类别色 + 网格中性色）
- [ ] 工具提示卡片化、图例可交互、0 轴清晰
- [ ] 支持阈值线/标注/区间高亮；复杂场景可补充 ECharts
- [ ] 图表在明/暗主题下对比度与网格可读性良好

## 8. 响应式与无障碍
- [ ] 桌面/平板/移动断点行为合理，无溢出/闪动
- [ ] 屏幕阅读器文本/aria-label 完备
- [ ] 键盘导航完整、焦点顺序合理

## 9. 性能与可维护性
- [ ] Tailwind 类名简洁、避免重复覆盖；使用语义变量
- [ ] 无无用依赖与样式；按需拆分组件，提高复用
- [ ] 一致的文件/组件命名与分层

## 10. 提交要求
- [ ] 提供明/暗主题 + 三端截图/录屏
- [ ] 变更说明、影响范围与回归点
- [ ] 遵循 PR 模板并完成自检清单
