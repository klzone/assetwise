# Step6 - UI CSS 审核与基线建立

目的
- 统一全局样式，建立可回归的视觉基线，降低后续回归成本。

现状要点
- 全局变量覆盖与主题切换一致性需进一步核对，部分组件在暗黑模式下对比度可能不一致。
- 部分页面的间距、字体大小、行高仍存在偏差，尤其在列表、表单、按钮等交互区域。
- 组件变体统一程度不足，个别页面直接使用局部样式，导致样式覆盖冲突。

工作计划
- 审核 globals.css、tailwind.config.js 的颜色、间距、圆角等 Token，与 CSS 变量保持一致。
- 将高频页面（资产列表、侧边栏、登录页、仪表盘）应用统一的变体与 Token。
- 记录缺陷清单，附上对比截图（现状 vs 期望）。
- 设计视觉基线截图存放结构，方便后续对比。

产出物
- ui-css-defects.md（缺陷清单及复现步骤）
- baseline_screenshots/ 目录及初步基线截图
- 扩展的 UI 测试用例（覆盖颜色对比、聚焦可见性、暗黑模式切换等）

下一步
- 根据缺陷清单提交修复摘要，更新后续 Plan。
